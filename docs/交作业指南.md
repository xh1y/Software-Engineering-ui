# OptiKG 项目交作业指南

## 一、你需要提交的文件总览

交作业总共需要 **2 个文件**：

| 文件 | 内容 | 大约大小 |
|------|------|----------|
| `OptiKG-源代码.zip` | 全部源代码 + 模型 + 文档 + 测试 | ~420MB |
| `optikg-image.tar.gz` | Docker 镜像（二进制交付物） | 取决于镜像大小 |

```

```

> ❗ **重要**：`tokenizers-cpp` 是 git submodule，直接打 zip 时子模块内容**不会被包含**。你需要在打 zip 之前确保这个目录有完整源码。检查方法：
>
> ```bash
> ls 3rdparty/tokenizers-cpp/CMakeLists.txt
> # 如果这个文件存在，说明源码已经拉下来了
> ```

### 2.6 ⚠️ 模型文件 `models/`（最容易被漏掉！）

```

models/model_fp32.onnx.data   ← 模型权重 (392MB) ← 最大文件！
models/metadata.json

models/tokenizer_config.json
models/vocab.txt
```

> ❗ **非常重要**：`.gitignore` 里写了 `*.onnx` 和 `*.onnx.data`，也就是说这些模型文件**不在 git 里**！你打 zip 时必须手动确认它们被包含进去。没有模型文件，程序跑不起来。

## 三、Docker 镜像提取（二进制交付物）

### 3.1 先构建镜像

```bash
cd /home/fed/Documents/clion-workspace/qt-try
docker build --network=host -f Dockerfile -t optikg:1.0 .
```

### 3.2 导出镜像

```bash
docker save optikg:1.0 | gzip > optikg-image.tar.gz
```

### 3.3 老师拿到后如何运行

```bash
docker load -i optikg-image.tar.gz
docker run -p 8080:8080 optikg:1.0
# 浏览器打开 http://localhost:8080 即可使用
```

## 四、一键打包命令

把下面的命令复制粘贴执行，一次性完成所有打包：

```bash
cd /home/fed/Documents/clion-workspace/qt-try

# ====== 1. 打包源码 ======
zip -r OptiKG-源代码.zip \
    CMakeLists.txt \
    .gitignore \
    .gitmodules \
    Dockerfile \
    .dockerignore \
    src/ \
    include/ \
    resources/ \
    models/ \
    onnxruntime-linux-x64-1.24.4.tgz \
    tests/ \
    test/ \
    qt-try_en.ts \
    qt-try_en.qm \
    docs/ \
    build.sh \
    run.sh \
    build-image.sh \
    build-appimage.sh \
    test.sh \
    .github/ \
    3rdparty/ \
    -x "3rdparty/tokenizers-cpp/.git"

# ====== 2. 验证模型文件是否包含 ======
unzip -l OptiKG-源代码.zip | grep -E "model_fp32|tokenizer\.json|vocab"

# ====== 3. 构建并导出 Docker 镜像 ======
docker build --network=host -f Dockerfile -t optikg:1.0 .
docker save optikg:1.0 | gzip > optikg-image.tar.gz

echo "====== 完成 ======"
ls -lh OptiKG-源代码.zip optikg-image.tar.gz
```

- [ ] `src/` 下面所有 `.cpp` 和 `.h` 都在
- [ ] `include/optikg/datamodel.h` 在
- [ ] `onnxruntime-linux-x64-1.24.4.tgz` 在
- [ ] `tests/` 下 14 个测试文件都在
- [ ] Docker 镜像导出成功且能正常 `docker load`
- [ ] 所有文档章节（1~5 章）的 `.md`/`.pdf` 都在
