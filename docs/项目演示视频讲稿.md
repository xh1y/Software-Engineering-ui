大家好，我们的项目是一个工业故障知识抽取系统
它能从维修日志、故障报告等非结构化文本中
自动抽取出部件、故障、工具、组成四类实体及其关联关系，生成可视化的知识图谱
整个过程完全离线运行，不需要联网
下面我来演示系统的完整使用流程
主界面采用四区域分割布局：
左上角是文本输入区，可以在这里输入或粘贴维修描述文本
右上角是知识图谱可视化区域，抽取出的实体和关系以力导向图的形式展示在这里
左下角是历史记录面板，所有抽取过的结果都会自动保存
右下角是结果展示，按实体分组展示抽取出的三元组
我们先来试一条简单的维修描述
输入：
"电机轴承出现异常磨损，使用振动分析仪检测到频率异常，轴承与转子配合间隙过大"
点击"开始抽取"，抽取完成后，右侧知识图谱自动渲染出实体节点和关系连线：
可以看到系统识别出了"电机轴承"、"磨损"、"间隙过大"等实体，并用连线标出了它们之间的关系
除了手动输入，也可以直接导入文本文件
菜单"文件→打开纯文本文件"，选择一个 .txt 文件，内容会自动加载到输入框中，然后就可以直接抽取了
下方结果展示按实体分组列出了所有三元组，每条都带有置信度分数
图谱支持丰富的交互操作：
按住鼠标左键拖拽节点，关联的连线会实时跟随
按住空白区域拖拽可以平移整个画布
按住 Ctrl 键配合鼠标滚轮可以进行缩放
如果觉得某个面板占地方，可以在视图菜单里切换面板的显示和隐藏
每次抽取的结果都会自动保存到 SQLite 数据库中
在左下角的历史记录面板可以看到所有的抽取历史
表格显示 ID、时间、处理耗时、平均置信度、实体数和关系数
可以通过关键词搜索历史记录——输入"轴承"，点击搜索，系统会筛选出包含"轴承"的记录
还可以按日期范围筛选：设置开始和结束日期
按实体类型过滤：下拉框可以选择全部、部件、故障、工具或组成
点击"清空筛选"恢复到默认显示
双击某条记录，系统会自动加载原文到输入框，同时图谱和结果也会更新为当时的抽取结果
如果需要清理旧数据：按住 Ctrl 多选，点击"批量删除"，确认后即可删除
删除操作是级联的，记录关联的实体和关系会一并清除
抽取完成后，可以把结果导出
点击菜单"文件→导出为JSON"，选择保存位置
导出的 JSON 格式包含版本号、导出时间、三元组数量，以及完整的三元组数组，同样可以导出为 CSV 格式，方便用 Excel 打开查看
CSV 表头包括主语名称、主语类型、关系、关系类型、宾语名称、宾语类型、置信度等完整信息
之前导出的 JSON 文件可以重新加载
菜单"文件→打开之前的结果"，选择 JSON 文件，图谱和结果树就会恢复当时的抽取结果
这样即使数据库清理了，重要的结果也能随时查看
如果需要处理大量文档，可以使用批量处理功能
点击工具栏"批量处理"
点击"添加文件"，选择多个 JSON 或 CSV 文件
系统支持标准 JSON 数组格式、JSON Lines 格式（每行一个 JSON 对象），以及 CSV 格式
在"字段映射"选项卡中，可以配置 JSON 内容字段名——默认是 text
CSV 内容列名和文件编码也可以在这里设置
点击"开始处理"
系统分两个阶段工作：首先扫描阶段统计总记录数，然后处理阶段逐条抽取
处理过程中进度条实时更新，日志区域显示每个文件的处理状态
处理是后台线程执行的，界面不会卡顿
处理完成，弹出统计报告，显示总文件数、成功数、失败数和总耗时
如果勾选了"自动导出"，处理结果会自动保存到指定的输出目录
点击工具栏"设置"或按 Ctrl+, 打开设置对话框
设置分为三个选项卡
第一个是"字段映射"——设置 JSON 内容字段名、CSV 内容列名和 CSV 编码，这些是批量处理时的默认配置
第二个是"模型设置"——可以指定 ONNX 模型文件路径，系统默认会自动检测 models 目录下的模型
置信度阈值范围从 -100 到 0，默认 -0.2
阈值越高，过滤越严格，只保留高置信度的结果
可以用步长 1 来调节
第三个是"批量处理"——设置默认的输出目录、导出格式，以及是否自动导出
如果不小心改乱了，点击"恢复默认"，确认后所有设置恢复到初始值
以上就是我们系统的全部内容， 感谢观看
